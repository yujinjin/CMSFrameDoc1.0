form 表单验证

form 表单验证是基于jquery的Validate第三方插件做的封装。它是一个高度灵活的工具，可以将任何HTML表格添加高级的交互功能。 本站点的封装重点是针对服务器数据请求让客户端表单验证变得更简单，同时提供了大量的定制选项，满足应用程序各种需求。该插件捆绑了一套有用的验证方法，包括 URL 和电子邮件验证，同时提供了一个用来编写用户自定义方法的 API。

其官方地址是：https://jqueryvalidation.org/

api地址是：https://jqueryvalidation.org/documentation/

github地址是：https://github.com/jquery-validation/jquery-validation

通过rules设置各个表单的验证规则，messages是表示表单验证的错误信息提示，如果某个控件没有 message，将调用默认的信息。

		            				<script>
define(["form-validate"], function (formValidate) {
	formValidate.validate("form[data-index=1]", {
		abpServiceProxiesFun: site.api.order.form,
		success: function(data) {
			site.toastr.success("操作成功！");
		}
	}, {
		rules: {
			"userName": {
				required: true,
				minlength: 6,
				maxlength: 20
			},
			"password": {
				required: true,
				minlength: 6,
				maxlength: 16
			},
			"repeat-password": {
				required: true,
				equalTo: "[name=password]"
			},
			"zipCode": {
				required: true,
				isZipCode: true
			},
			"select": {
				required: true
			},
			"radio": {
				required: true
			},
			"checkbox": {
				required: true,
				minlength: 2
			},
			"remark": {
				required: true
			}
		},
		messages: {
			"userName": {
				required: "请输入用户名",
				minlength: "用户名至少6个字符",
				maxlength: "用户名最多20个字符"
			},
			"password": {
				required: "请输入密码",
				minlength: "用户名至少6个字符",
				maxlength: "用户名最多16个字符"
			},
			"repeat-password": {
				required: "请输入确认密码!",
				equalTo: "两次密码输入不一致"
			},
			"zipCode": {
				required: "请输入邮编",
				isZipCode: "请输入正确的邮编格式！"
			}
		}
	});
});
</script>

添加自定义校验

见代码

通过addMethod(name,method,message)方法来添加除内置的 Validation 方法之外的验证方法。

参数 name 是添加的方法的名字。

参数 method 是一个函数，接收三个参数 (value,element,param) 。

value 是元素的值，element 是元素本身，param 是参数。

		            				<script>
// 中文字两个字节
$.validator.addMethod("byteRangeLength", function(value, element, param) {
    var length = value.length;
    for(var i = 0; i < value.length; i++){
        if(value.charCodeAt(i) > 127){
            length++;
        }
    }
  return this.optional(element) || ( length >= param[0] && length <= param[1] );   
}, $.validator.format("请确保输入的值在{0}-{1}个字节之间(一个中文字算2个字节)"));

// 邮政编码验证   
$.validator.addMethod("isZipCode", function(value, element) {   
    var tel = /^[0-9]{6}$/;
    return this.optional(element) || (tel.test(value));
}, "请正确填写您的邮政编码");
</script>

Options

下面是常用的表单验证常用配置参数，如果还需要更多请查看jquery.validate的官方文档。

序号 参数 说明 类型 可选值 默认值
1 debug 进行调试模式 boolean — false
2 submitHandler 通过验证后运行的函数，里面要加上表单提交的函数，否则表单不会提交。 function — —
3 ignore 对某些元素不进行验证。 string — ":hidden"
4 rules 自定义规则，key:value 的形式，key 是要验证的元素，value 可以是字符串或对象。 object — —
5 messages 自定义的提示信息，key:value 的形式，key 是要验证的元素，value 可以是字符串或函数。 object — —
6 groups 对一组元素的验证，用一个错误提示，用 errorPlacement 控制把出错信息放在哪里。 object — —
7 groups 对一组元素的验证，用一个错误提示，用 errorPlacement 控制把出错信息放在哪里。 object — —
8 OnSubmit 指定是否提交时验证。 boolean — true
9 onfocusout 指定是否在获取焦点时验证。 boolean — true
10 onkeyup 指定是否在敲击键盘时验证。 boolean — true
11 onclick 指定是否在鼠标点击时验证（一般验证 checkbox、radiobox）。 boolean — true
12 focusInvalid 提交表单后，未通过验证的表单（第一个或提交之前获得焦点的未通过验证的表单）会获得焦点。 boolean — true
13 focusCleanup 当未通过验证的元素获得焦点时，移除错误提示（避免和 focusInvalid 一起使用）。 boolean — false
14 errorClass 指定错误提示的 css 类名，可以自定义错误提示的样式。 string — error
15 errorElement 指定使用什么标签标记错误。 string — label
16 wrapper 指定使用什么标签再把上边的 errorELement 包起来。 string — —
17 errorLabelContainer 把错误信息统一放在一个容器里面。 Selector(选择器) — —
18 showErrors 可以显示总共有多少个未通过验证的元素。 function — —
19 errorPlacement 可以自定义错误放到哪里。 function — —
20 success 要验证的元素通过验证后的动作，如果跟一个字符串，会当作一个 css 类，也可跟一个函数。 function|string — —
21 highlight 可以给未通过验证的元素加效果、闪烁等。 function — —
